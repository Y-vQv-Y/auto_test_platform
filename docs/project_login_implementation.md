# 在项目中实现网站登录的详细指南

针对您提出的“在项目中怎么实现登录网站”的问题，本平台提供了三种互补的方案，您可以根据目标网站的复杂程度选择最合适的一种。

## 方案一：全自动登录（推荐）

适用于没有复杂验证码（如拼图、短信验证码）的网站。

1.  **配置凭据**：在项目详情页点击紫色 CPU 图标。
2.  **填写选择器**：提供用户名输入框、密码输入框和登录按钮的 CSS 选择器。
3.  **平台执行**：平台会自动打开浏览器，填写信息并点击登录，随后自动抓取并保存 Cookie。

**优点**：全自动，无需人工干预。
**缺点**：无法处理复杂的图形验证码。

## 方案二：手动辅助登录（处理验证码）

适用于有滑块、拼图或点击验证码的网站。

1.  **触发登录**：点击“登录状态”卡片中的手机图标。
2.  **人工介入**：平台会弹出一个可见的浏览器窗口。
3.  **完成操作**：您在弹出的浏览器中手动输入账号密码并完成验证码校验。
4.  **自动抓取**：一旦您登录成功，平台会检测到页面变化，自动抓取当前的 Cookie 和 LocalStorage 并保存。

**优点**：能处理任何复杂的登录验证。
**缺点**：需要人工操作一次。

## 方案三：手动导入 Cookie

适用于上述方案均受限，或您已经拥有有效 Cookie 的情况。

1.  **提取 Cookie**：按照[《Cookie 提取指南》](cookie_extraction_guide.md)中的方法，从您自己的浏览器中获取 JSON 格式的 Cookie 数组。
2.  **粘贴保存**：在项目详情页点击“登录状态”卡片，选择“手动保存登录态”，将 JSON 粘贴进去。

**优点**：灵活性最高。
**缺点**：受 `HttpOnly` 限制，手动提取可能不完整。

---

## 核心机制：登录态校验与自动恢复

无论您使用哪种方案，平台都具备以下核心能力：

1.  **有效性校验**：在每次执行自动化测试前，平台都会先加载保存的 Cookie 访问目标网站，检查是否能看到“个人中心”或“退出”等标志。
2.  **自动恢复**：在 AI 生成的测试脚本中，平台会自动注入 `storage_state`。这意味着您的测试脚本不需要写任何登录代码，打开页面即是登录状态。
3.  **状态反馈**：前端界面会实时显示登录态是否有效（绿色表示有效，红色表示失效），您可以随时点击“刷新”按钮重新获取登录态。
