# 自动登录配置指南

## 1. 功能简介

在自动化测试中，处理需要登录的网站是一个常见的挑战。虽然手动提取 Cookie 是一种方法，但它容易受到 `HttpOnly` 限制且操作繁琐。本平台提供的“自动登录”功能允许您配置登录凭据和页面元素选择器，由平台自动执行登录过程并保存会话信息。

## 2. 配置步骤

### 2.1 获取元素选择器

在配置自动登录之前，您需要获取目标网站登录页面的元素选择器：

1.  打开目标网站的登录页面。
2.  按 `F12` 打开开发者工具。
3.  使用“元素选择器”工具（通常是左上角的小箭头图标）点击用户名输入框、密码输入框和登录按钮。
4.  记录下它们的唯一标识符，例如：
    *   **用户名选择器**：`#username` 或 `input[name="user"]`
    *   **密码选择器**：`#password` 或 `input[type="password"]`
    *   **登录按钮选择器**：`button[type="submit"]` 或 `.login-btn`

### 2.2 在平台中配置

1.  进入项目详情页。
2.  在“登录状态”卡片中，点击紫色的小图标（CPU 图标）打开“自动登录配置”模态框。
3.  填写以下信息：
    *   **登录页面 URL**：登录页面的完整地址。
    *   **用户名选择器**：步骤 2.1 中获取的用户名输入框选择器。
    *   **密码选择器**：步骤 2.1 中获取的密码输入框选择器。
    *   **登录按钮选择器**：步骤 2.1 中获取的登录按钮选择器。
    *   **用户名**：您的登录账号。
    *   **密码**：您的登录密码（平台会加密存储）。
4.  点击“保存并测试登录”。

## 3. 工作原理

1.  **自动执行**：平台会启动一个无头浏览器，导航到您配置的 URL。
2.  **模拟操作**：自动填写用户名和密码，并点击登录按钮。
3.  **状态识别**：登录后，平台会智能识别页面内容（如寻找“退出登录”、“个人中心”等关键字或元素）来确认登录是否成功。
4.  **会话保存**：登录成功后，平台会自动提取所有的 Cookie 和 LocalStorage 数据，并保存到数据库中。
5.  **测试集成**：在后续执行自动化测试时，平台会自动加载这些保存的会话信息，实现免登录测试。

## 4. 常见问题

*   **登录失败**：请检查选择器是否正确，以及用户名密码是否有效。如果网站有复杂的验证码（如拼图、点击文字等），自动登录可能会失败，此时建议使用“手动登录（处理验证码）”功能。
*   **登录态失效**：您可以随时点击“校验登录态”按钮来检查当前会话是否仍然有效。如果失效，可以再次触发自动登录或手动登录。
